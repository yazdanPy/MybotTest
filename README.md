# 🤖 ربات حساب‌کتاب دوستانه (Hesabkitab Bot)

ربات تلگرامی برای مدیریت حساب‌کتاب دورهمی‌های گروه دوستان: هرکی هرجا رو حساب کرد ثبت می‌کنه،
ربات با توجه به تعداد نفرات هر کسی (خودش + همراهانش) هزینه رو تقسیم می‌کنه، پرداختی‌ها رو ثبت
می‌کنه و همه‌چیز رو **تهاتر** می‌کنه تا در نهایت فقط یه بدهی خالص و ساده بین هر دو نفر بمونه.

## ✨ امکانات

- عضویت اعضا با تایید مدیر گروه؛ هر عضو تعداد نفرات (وزن خانواده) و شماره کارت خودش رو ثبت می‌کنه.
- مدیر می‌تونه عضو اضافه/حذف کنه، تعداد نفرات یا شماره کارت هرکس رو ویرایش کنه، یا یه نفر دیگه رو هم مدیر کنه.
- ثبت هزینه جدید توسط هرکسی که پرداخت کرده: مبلغ، موضوع خرید، و اینکه چه کسانی حضور داشتن.
- تقسیم هزینه به یکی از این چهار روش:
  - بر اساس تعداد نفرات هرکس (پیش‌فرض)
  - مساوی بین همه
  - **مبلغ دلخواه برای هرکس** (مثلاً تو رستوران که قیمت غذاها فرق داره، برای هرکس جدا مبلغ وارد می‌کنی)
  - **استخراج خودکار از عکس رسید با هوش مصنوعی Veryfi** (اختیاری، نیاز به تنظیم داره -- پایین‌تر توضیح دادم): عکس فاکتور رو می‌فرستی، ربات آیتم‌ها رو در میاره و برای هرکدوم می‌پرسه مال کیه.
- **تعداد نفرات قابل تغییر برای هر هزینه**: تعداد نفرات پیش‌فرض هرکس از پروفایلشه، ولی موقع
  ثبت هر هزینه می‌تونی برای همون یه بار مشخص کنی مثلاً امروز فلانی با ۲ نفر بوده، بدون اینکه
  پروفایل دائمیش تغییر کنه (چون تعداد همراهان آدم‌ها هر بار ممکنه فرق کنه).
- **رسید اختیاری**: بعد از ثبت هر هزینه یا پرداخت، می‌تونی یه عکس از رسید/فیش بانکی یا یه توضیح
  متنی ضمیمه کنی؛ بعداً از تاریخچه قابل مشاهده‌ست.
- امکان تغییر «پرداخت‌کننده» در لحظه ثبت (مثلاً اگه یکی دیگه به جای خودت وارد کرده).
- **تهاتر خودکار**: اگه امروز یکی حساب کنه و قبلاً طرف مقابل یه جای دیگه حساب کرده بوده، ربات
  فقط تفاوت خالص رو به عنوان بدهی نشون می‌ده، نه هر تراکنش رو جدا جدا.
- **پیشنهاد تسویه گروهی**: با یه الگوریتم ساده (همون روشی که اپ‌های تقسیم‌هزینه معروف استفاده می‌کنن)
  کمترین تعداد واریزی که کل گروه رو صاف می‌کنه رو پیشنهاد می‌ده.
- ثبت پرداخت (کارت‌به‌کارت) با **تایید گیرنده** -- یعنی تا طرف مقابل تایید نکنه که پول رو گرفته،
  توی حساب‌ها لحاظ نمی‌شه. این از دعوا و اختلاف جلوگیری می‌کنه.
- گزارش وضعیت شخصی («📊 وضعیت حساب من») با شماره کارت طرف مقابل (یه ضربه بزن تا کپی بشه).
- تاریخچه‌ی کامل و ریز هر هزینه (تاریخ، مبلغ، موضوع، سهم تک‌تک افراد) با امکان صفحه‌بندی و حذف
  (توسط مدیر یا ثبت‌کننده‌ی همون هزینه).
- گزارش کامل گروه برای مدیر + **خروجی اکسل** (سه شیت: هزینه‌ها، پرداخت‌ها، وضعیت فعلی).
- همه‌ی تاریخ‌ها شمسی (جلالی) هستن و مبلغ‌ها به تومان.
- اطلاع‌رسانی خودکار به گروه تلگرامی‌تون بعد از هر هزینه جدید (با `/setgroup`).
- منوی دکمه‌ای پایین صفحه، برای کسایی که حوصله‌ی به‌خاطر سپردن دستورات رو ندارن.

## 📋 پیش‌نیازها

- یک عدد **توکن ربات تلگرام**: به [@BotFather](https://t.me/BotFather) پیام بده، `/newbot` رو بزن
  و مراحلش رو طی کن تا یه توکن مثل `123456789:AAxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` بگیری.
- Python نسخه 3.10 یا بالاتر (یا Docker، اگه ترجیح می‌دی).
- یه جایی که ربات بتونه ۲۴ ساعته روشن بمونه: یه سرور مجازی (VPS)، یه Raspberry Pi خونگی، یا حتی
  لپ‌تاپ خودت اگه همیشه روشنه.

> ⚠️ **نکته مهم برای کاربران ایرانی:** چون تلگرام در ایران فیلتره، بهتره ربات رو روی یک VPS خارج
> از ایران اجرا کنید تا بدون مشکل به سرورهای تلگرام وصل بشه.

## 🚀 نصب و راه‌اندازی (روش معمولی)

```bash
# ۱) وارد پوشه پروژه شو
cd hesabkitab_bot

# ۲) (پیشنهادی) یک محیط مجازی پایتون بساز
python3 -m venv .venv
source .venv/bin/activate      # در ویندوز: .venv\Scripts\activate

# ۳) پکیج‌های لازم رو نصب کن
pip install -r requirements.txt

# ۴) فایل تنظیمات بساز
cp .env.example .env
# حالا فایل .env رو باز کن و BOT_TOKEN رو با توکن واقعی‌ات جایگزین کن

# ۵) ربات رو اجرا کن
python bot.py
```

اگه همه‌چیز درست باشه، توی ترمینال می‌بینی: `Starting bot polling...`

## 🐳 نصب و راه‌اندازی با Docker (جایگزین)

```bash
cp .env.example .env
# .env رو ویرایش کن و BOT_TOKEN رو بذار
docker compose up -d --build
```

دیتابیس داخل پوشه‌ی `./data` کنار پروژه ذخیره می‌مونه (حتی اگه کانتینر رو حذف کنی، دیتات می‌مونه).

## 🏁 اولین قدم‌ها بعد از روشن شدن ربات

1. توی تلگرام، ربات رو پیدا کن (با همون یوزرنیمی که موقع ساخت در BotFather دادی) و `/start` رو بزن.
   **اولین نفری که این کارو بکنه، خودکار مدیر گروه می‌شه** (چون هنوز مدیری وجود نداره).
2. لینک ربات رو به بقیه دوستات بده. هرکدوم باید `/start` بزنن و شماره کارت + تعداد نفراتشون
   (خودشون + همراهانی مثل همسر/فرزند) رو وارد کنن.
3. تو (مدیر) با دستور یا دکمه‌ی «⏳ درخواست‌های در انتظار» درخواست‌هاشون رو تایید کن.
4. اگه می‌خوای خلاصه‌ی هر هزینه خودکار توی گروه تلگرامی‌تون هم فرستاده بشه: ربات رو به اون گروه
   اضافه کن و داخل خود گروه دستور `/setgroup` رو بزن (باز هم فقط مدیر می‌تونه).
5. از این به بعد، هرکی جایی رو حساب کرد، از دکمه‌ی «➕ ثبت هزینه جدید» استفاده کنه!

## 🧾 فعال کردن استخراج خودکار رسید با هوش مصنوعی (Veryfi) -- اختیاری

این قابلیت کاملاً اختیاریه؛ بدون تنظیمش هم بقیه ربات عادی کار می‌کنه.

1. برو به [hub.veryfi.com](https://hub.veryfi.com/) و یه حساب بساز (پلن رایگانشون برای شروع کافیه).
2. بعد از ورود، برو به بخش **Settings -> Keys** و این سه مقدار رو بردار: **Client ID**، **Username**، **API Key**.
3. این سه‌تا رو توی فایل `.env` بذار:
   ```
   VERYFI_CLIENT_ID=...
   VERYFI_USERNAME=...
   VERYFI_API_KEY=...
   ```
4. ربات رو دوباره اجرا کن (یا روی Railway، دوباره دیپلوی کن). حالا موقع ثبت هزینه، گزینه‌ی
   «🧾 استخراج خودکار از عکس رسید (AI)» هم توی لیست روش‌های تقسیم ظاهر می‌شه.

> این بخش از کد بر اساس مستندات رسمی Veryfi نوشته شده ولی چون به یه حساب واقعی Veryfi
> دسترسی نداشتم، نتونستم با یه رسید واقعی امتحانش کنم. اگه پیغام خطای غیرمنتظره‌ای دیدی
> (مثلاً یه فیلد جدید که مستندات Veryfi عوض شده)، متن خطا رو برام بفرست تا اصلاحش کنم.

## 🧮 تقسیم هزینه و تهاتر چطور کار می‌کنه؟

- هر عضو یه «تعداد نفرات» پیش‌فرض داره (مثلاً علی چون همسر و بچه هم میاره، عدد ۳ رو ثبت کرده؛ رضا چون
  تنها میاد، عدد ۱). این فقط یه **پیش‌فرض** برای راحتی‌شه.
- موقع ثبت هر هزینه، بعد از انتخاب «تقسیم بر اساس تعداد نفرات»، می‌تونی برای همون هزینه‌ی خاص
  تعداد نفرات هرکس رو با دکمه‌های ➖ ➕ عوض کنی (مثلاً امروز محراب ۲ نفر بوده، امیر ۴ نفر) --
  این فقط برای همون هزینه اعمال می‌شه و پروفایل دائمی کسی رو تغییر نمی‌ده.
- وقتی هزینه‌ای ثبت می‌شه، فقط بین کسایی که آن بار حضور داشتن تقسیم می‌شه -- بر اساس همون تعداد نفراتی
  که برای همون هزینه مشخص کردی. مثلاً یه هزینه‌ی ۴۰۰,۰۰۰ تومانی بین علی (۳ نفر) و رضا (۱ نفر) می‌شه
  ۳۰۰,۰۰۰ و ۱۰۰,۰۰۰.
- اگه هزینه‌هاتون قیمت متفاوت داره (مثلاً رستوران)، به‌جای این محاسبه، «مبلغ دلخواه برای هرکس» رو
  انتخاب کن و مبلغ دقیق سهم هرکس رو خودت وارد کن؛ جمع کل خودکار از روی همین اعداد حساب می‌شه.
- ربات به ازای هر هزینه، بدهی «فلانی به پرداخت‌کننده» رو توی یک دفترچه‌ی داخلی ثبت می‌کنه.
- هر پرداختی (بعد از تایید گیرنده) از این بدهی کم می‌شه.
- در هر لحظه، بین هر دو نفر فقط **تفاوت خالص** (تهاتر شده) نمایش داده می‌شه -- نه کل تاریخچه.
  یعنی اگه هفته پیش رضا ۱۰۰ به علی بدهکار شده بود و امروز علی ۱۵۰ به رضا بدهکار شد، ربات
  می‌گه: «علی ۵۰ به رضا بدهکاره»، نه اینکه دو تا بدهی جدا نشون بده.
- علاوه بر این، «گزارش کامل گروه» یه پیشنهاد هم می‌ده که با کمترین تعداد واریزی، کل بدهی‌های
  گروه (نه فقط دو نفر دو نفر) صاف بشه.

## 🔌 مشکل اتصال به تلگرام؟ (`TimedOut` یا مشابه)

اگه موقع اجرا خطای `telegram.error.TimedOut` یا شبیهش گرفتی، یعنی سیستمی که ربات رو
روش اجرا می‌کنی نمی‌تونه به سرورهای تلگرام وصل بشه -- معمولاً چون از داخل ایران و
بدون فیلترشکن فعال اجرا شده (خود تلگرام، نه فقط اپش، در سطح شبکه فیلتره).

- **بهترین راه‌حل:** ربات رو روی یه VPS خارج از ایران اجرا کن (چون این ربات باید ۲۴
  ساعته روشن بمونه، به هر حال یه سرور بیرون از ایران لازم داری).
- **اگه فقط می‌خوای موقتاً روی سیستم خودت تست کنی** و از ابزاری مثل v2rayN / V2Box /
  Shadowrocket / ClashX استفاده می‌کنی که یه پراکسی محلی بالا میاره: آدرس اون پراکسی
  رو در `.env` توی متغیر `PROXY_URL` بذار (مثال داخل `.env.example` هست) و ربات رو
  دوباره اجرا کن. نیازی به تغییر کد نیست.

## 💾 پشتیبان‌گیری از اطلاعات

همه‌ی اطلاعات (اعضا، هزینه‌ها، پرداخت‌ها) توی یک فایل ساده به اسم `hesabkitab.db`
(یا هرچی که در `.env` گذاشتی) ذخیره می‌شه. برای پشتیبان‌گیری، کافیه همین یک فایل رو کپی
کنی و جایی نگه داری -- هر از گاهی این کارو بکن.

## 🔒 اجرای همیشگی روی سرور (بدون Docker)

اگه از Docker استفاده نمی‌کنی و می‌خوای ربات روی یک VPS همیشه روشن بمونه، از فایل نمونه‌ی
`hesabkitab-bot.service.example` استفاده کن (راهنمای کامل داخل خود فایل نوشته شده) تا با
`systemd` به‌صورت خودکار اجرا و در صورت کرش، ری‌استارت بشه.

## 🧪 تست‌ها

منطق اصلی (تقسیم هزینه، تهاتر، دیتابیس، و کل جریان ربات) از قبل با تست‌های خودکار بررسی
شده. اگه خواستی بعد از تغییر دادن کد دوباره مطمئن بشی چیزی خراب نشده:

```bash
python test_split_engine.py   # منطق تقسیم و تهاتر
python test_database.py       # دیتابیس و محاسبه‌ی موجودی‌ها
python test_migration.py      # امن بودن آپدیت روی دیتابیسی که از قبل دیتا داره
python test_veryfi_client.py  # پردازش پاسخ Veryfi (با پاسخ شبیه‌سازی‌شده)
python test_bot_flow.py       # کل جریان ربات (عضویت تا گزارش) بدون نیاز به اتصال واقعی تلگرام
```

## 🔮 محدودیت‌های فعلی / ایده برای توسعه بعدی

- این نسخه برای **یک** جمع دوستانه طراحی شده (نه چند گروه مستقل روی یک ربات). اگه خواستی برای
  چند جمع جدا از هم استفاده کنی، برای هرکدوم یک نمونه‌ی جدا از ربات (توکن و دیتابیس جدا) بساز.
- دسته‌بندی هزینه‌ها (مثلاً «رستوران»، «سفر») پیاده نشده؛ اگه خواستی راحت می‌شه اضافه کرد.
- اگه دوست داری، می‌تونم یادآوری خودکار هفتگی برای بدهی‌های باز، یا آمار «کی بیشتر خرج کرده»
  رو هم بهش اضافه کنم -- کافیه بگی.
